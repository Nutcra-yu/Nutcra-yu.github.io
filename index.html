布局 Layout

官方定义

布局定义了应用中的界面结构。
布局中的所有元素均使用 View 和 ViewGroup 对象的层次结构进行构建。

View 通常用于绘制用户可看到并与之交互的内容。
ViewGroup 则是不可见的容器，用于定义 View 和其他 ViewGroup 对象的布局结构

dp
0dp(又叫match_constraint)

sp

二、View Group

（主要写 constrain layout）

constrain layout

(1).控件的大小

控件的大小无非就两个属性 ：宽 weight 和 高 height

接下来讲三个API：

default

用于设置控件的大小属性

View的宽度定义为0dp的时候才生效，其余情况下设置这个属性是不起任何作用的。

app:layout_constraintWidth_default有三个值：wrap、spread和percent。

  1. wrap：等价于 wrap_content

  2. spread：等价于match_parent

  3. percent`：

    设置View的宽度为parent的比例值，比例值默认是100%，即match_parent。

    这个比例值通过属性app:layout_constraintWidth_percent设置。

percent

用于设置控件的大小

值范围是0-1，默认是1即100%，表示View的宽高占parent的比例值。

Dimension Ratio

用于设置宽高比

Dimension : n. 大小，尺寸；（空间的）维度；范围，规模；方面，部分
Ratio : n . 比率，比例

设置宽高比,宽高中必须有一个是0dp，而必须是可计算出的值。

例如：app:layout_constraintDimensionRatio="4:9"

表示 宽比高 为 4：9

(2).控件的位置

首先需要知道几个名词

top：顶部
bottom：底部
left、start：左边
right、end：右边
baseline：多数字母排列的基准线
( 同样表示左边的 left和start ，和同样表示右边的right和end，有什么区别呢？详细见附录)

API解释

··· to ···

用于对齐边界

比如 top to top 就是顶部对着顶部

A top to top B

top to bottom 就是顶部对着底部

A top to bottom B

start 和 end 也同理

bias

用于设置控件所处的比例

Horizontal :adj. 水平的
Vertical :adj. 竖直的

  • 使用前提

    必须先设置该方向上的两个约束

  • 比例说明

    水平方向 就是左半部分 比上 左右空白部分之和

    竖直方向 就是上半部分 比上 上下空白部分之和

    以 水平方向 Horizontal_bias 为例
    $$
    bias = 100 / (100+200) ≈ 0.33
    $$

    以 竖直方向 Vertical_bias 为例
    $$
    bias = 150 / (150+500) ≈ 0.23
    $$

left start？rtght end？

Left 是指左侧，Start 是指开头

世界上有两种语言阅读顺序，一种是从左到右 LTR(Left To Right) ,还有一种是从右向左 RTL(Right To Left)

一般情况下，View的开始部分就是左边，但比如阿拉伯语这种RTL语言，为了支持从右往左的布局，Android从4.2开始推荐使用start、end来代替left、right

res 资源

官方定义

目录 资源类型
animator/ 用于定义属性动画的 XML 文件。
anim/ 用于定义渐变动画的 XML 文件。（属性动画也可保存在此目录中，但为了区分这两种类型，属性动画首选 animator/ 目录。）
color/ 用于定义颜色状态列表的 XML 文件。请参阅颜色状态列表资源
drawable/ 位图文件（.png、.9.png、.jpg、.gif）或编译为以下可绘制对象资源子类型的 XML 文件：位图文件九宫格（可调整大小的位图）状态列表形状动画可绘制对象其他可绘制对象请参阅 Drawable 资源。
mipmap/ 适用于不同启动器图标密度的可绘制对象文件。如需了解有关使用 mipmap/ 文件夹管理启动器图标的详细信息，请参阅管理项目概览。
layout/ 用于定义用户界面布局的 XML 文件。请参阅布局资源。
menu/ 用于定义应用菜单（如选项菜单、上下文菜单或子菜单）的 XML 文件。请参阅菜单资源。
raw/ 需以原始形式保存的任意文件。如要使用原始 InputStream 打开这些资源，请使用资源 ID（即 R.raw.*filename*）调用 Resources.openRawResource()。但是，如需访问原始文件名和文件层次结构，则可以考虑将某些资源保存在 assets/ 目录（而非 res/raw/）下。assets/ 中的文件没有资源 ID，因此您只能使用 AssetManager 读取这些文件。
values/ 包含字符串、整型数和颜色等简单值的 XML 文件。其他 res/ 子目录中的 XML 资源文件会根据 XML 文件名定义单个资源，而 values/ 目录中的文件可描述多个资源。对于此目录中的文件，<resources> 元素的每个子元素均会定义一个资源。例如，<string> 元素会创建 R.string 资源，<color> 元素会创建 R.color 资源。由于每个资源均使用自己的 XML 元素进行定义，因此您可以随意命名文件，并在某个文件中放入不同的资源类型。但是，您可能需要将独特的资源类型放在不同的文件中，使其一目了然。例如，对于可在此目录中创建的资源，下面给出了相应的文件名约定：arrays.xml：资源数组（类型数组）。colors.xml：颜色值。dimens.xml：尺寸值。strings.xml：字符串值。styles.xml：样式。请参阅字符串资源、样式资源和更多资源类型。
